# 개발 (Development)

## 개발 환경 설정

### 필수 도구
- **JDK**: Java 21
- **Kotlin**: 1.9+
- **Node.js**: 18+ / npm 또는 yarn
- **Git**: 버전 관리
- **IDE**: IntelliJ IDEA (백엔드) / VS Code (프론트엔드)

### 로컬 환경 구성
```bash
# 백엔드
git clone <repo>
cd backend
./gradlew clean build
./gradlew bootRun

# 프론트엔드
cd frontend
npm install
npm start
```

## 코딩 컨벤션

### Kotlin (Backend)
- **명명규칙**: camelCase (변수, 함수), PascalCase (클래스)
- **들여쓰기**: 4 spaces
- **코드 스타일**: Kotlin Official Code Style
- **타입**: 명시적 타입 선언 권장

### JavaScript/TypeScript (Frontend)
- **명명규칙**: camelCase (변수, 함수), PascalCase (컴포넌트)
- **들여쓰기**: 2 spaces
- **세미콜론**: 필수
- **타입**: TypeScript 사용 권장

## 개발 프로세스

### 1. 기능 개발 플로우
```
1. 요구사항 분석
2. 설계 문서 작성
3. 브랜치 생성 (feature/기능명)
4. 코드 작성 및 테스트
5. Pull Request 생성
6. 코드 리뷰
7. 병합 (Merge)
8. 배포
```

### 2. 브랜치 전략 (Git Flow)
- `main`: 프로덕션 코드
- `develop`: 개발 브랜치
- `feature/*`: 기능 개발 브랜치
- `hotfix/*`: 긴급 수정 브랜치

## 테스트 전략

### Backend (Kotlin)
- **단위 테스트**: JUnit 5, Mockito
- **통합 테스트**: Spring Boot Test
- **커버리지**: 80% 이상 목표
- **테스트 실행**: `./gradlew test`

### Frontend (React)
- **단위 테스트**: Jest, React Testing Library
- **E2E 테스트**: Cypress / Playwright
- **테스트 실행**: `npm test`

## 문서화
- **API 문서**: Swagger/OpenAPI
- **코드 주석**: 필요한 경우에만
- **README**: 프로젝트 개요, 설치 방법, 사용법

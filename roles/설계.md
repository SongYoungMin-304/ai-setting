# 설계 (Design)

## 시스템 아키텍처
```
[React Frontend] <--> [REST API] <--> [Kotlin Spring Boot Backend] <--> [Database]
```

## 백엔드 아키텍처
- **패턴**: Layered Architecture (Controller → Service → Repository → Entity)
- **프로젝트 구조**:
  ```
  src/
  ├── main/kotlin
  │   └── com/project
  │       ├── controller/
  │       ├── service/
  │       ├── repository/
  │       ├── domain/
  │       └── config/
  └── test/
  ```

## 프론트엔드 아키텍처
- **패턴**: Component-based Architecture
- **디렉터리 구조**:
  ```
  src/
  ├── components/
  ├── pages/
  ├── services/
  ├── hooks/
  ├── contexts/
  ├── utils/
  └── styles/
  ```

## 데이터베이스 설계
### 주요 테이블
- `users`: 사용자 정보
- `posts`: 게시물 (예시)
- `comments`: 댓글 (예시)

### 관계도
- User 1 : N Posts
- Post 1 : N Comments

## API 엔드포인트 규칙
- **Base URL**: `/api/v1`
- **인증**: JWT 토큰
- **응답 형식**: JSON
- **에러 처리**: Standard HTTP Status Codes

### 주요 엔드포인트 예시
- `POST /api/v1/auth/login` - 로그인
- `GET /api/v1/users/{id}` - 사용자 조회
- `POST /api/v1/posts` - 게시물 작성
- `GET /api/v1/posts` - 게시물 목록

## 통신 흐름
1. 프론트엔드에서 HTTP 요청 → 백엔드 API
2. 백엔드는 요청 검증 → 비즈니스 로직 처리 → DB 조회/저장
3. 응답을 JSON으로 반환
4. 프론트엔드에서 상태 업데이트 및 UI 렌더링

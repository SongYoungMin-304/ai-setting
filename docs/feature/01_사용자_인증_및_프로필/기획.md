# 기능: 사용자 인증 및 프로필 관리 - 기획 문서

## 1. 기능 개요

### 목표
사용자가 안전하게 계정을 생성하고, 로그인/로그아웃하며, 프로필을 관리할 수 있는 기능 제공

### 우선순위
**HIGH** - 모든 다른 기능의 기반이 되는 필수 기능

### 예상 기간
1주일 (기획, 설계, 개발, 테스트 포함)

---

## 2. 요구사항

### 2.1 회원가입
- [ ] 이메일 주소로 가입
- [ ] 비밀번호 설정 (최소 8자, 숫자+특수문자 포함)
- [ ] 사용자명 설정 (3-20자, 영문/숫자만 가능)
- [ ] 이메일 중복 확인
- [ ] 사용자명 중복 확인
- [ ] 약관 동의 필수
- [ ] 가입 성공 시 자동 로그인

### 2.2 로그인
- [ ] 이메일 + 비밀번호로 인증
- [ ] JWT 토큰 발급 (Access Token + Refresh Token)
- [ ] 로그인 실패 시 에러 메시지 표시
- [ ] 계정 잠금 기능 (5회 실패 시 10분 잠금)

### 2.3 로그아웃
- [ ] 로그아웃 시 Refresh Token 무효화
- [ ] Access Token 제거

### 2.4 프로필 관리
- [ ] 프로필 조회 (사용자 정보, 가입일, 게시물 수, 댓글 수)
- [ ] 프로필 수정 (프로필 사진, 자기소개, 닉네임)
- [ ] 비밀번호 변경

### 2.5 메인 페이지 게시글 리스트 (공개)
- [ ] 로그인 없이 게시글 리스트 조회 가능
- [ ] 페이지네이션 (기본 10개 per page)
- [ ] 게시글: 제목, 작성자, 작성일, 조회수, 댓글 수 표시
- [ ] 게시글 작성 버튼 (로그인 필요)
- [ ] 게시글 상세 조회 (로그인 필요 없음)
- [ ] 정렬: 최신순, 인기순 (댓글 수/조회수)

### 2.6 추가 기능
- [ ] 비밀번호 재설정 (이메일 인증)
- [ ] 계정 탈퇴
- [ ] 사용자 이미지 업로드 (프로필 사진)

## 3. 사용자 스토리

### US-001: 회원가입
```
AS A 신규 사용자
I WANT TO 이메일과 비밀번호로 가입하기
SO THAT 사이트를 이용할 수 있다
```

**수용 기준:**
- 유효한 이메일 형식 검증
- 비밀번호 보안 요구사항 충족
- 중복 가입 방지
- 가입 후 자동 로그인

### US-002: 로그인
```
AS A 기존 사용자
I WANT TO 이메일과 비밀번호로 로그인하기
SO THAT 내 계정에 접근할 수 있다
```

**수용 기준:**
- JWT 토큰 발급
- 보안 쿠키에 저장
- 자동 로그인 옵션

### US-003: 프로필 조회
```
AS A 로그인한 사용자
I WANT TO 내 프로필을 볼 수 있고 다른 사용자의 프로필을 볼 수 있기
SO THAT 사용자 정보를 확인할 수 있다
```

## 4. 기술 스택

### Backend
- **Framework**: Spring Boot 3.x
- **ORM**: JPA / Hibernate
- **Database**: H2 (개발), PostgreSQL (운영)
- **인증**: JWT (Spring Security)
- **패스워드 암호화**: BCrypt
- **이미지 처리**: MultipartFile

### Frontend
- **Library**: React 18+
- **상태관리**: Context API / Redux
- **HTTP**: Axios
- **폼 처리**: React Hook Form
- **유효성 검사**: Zod / Yup

## 5. 데이터 모델

### User 엔티티
```
- id: Long (PK)
- email: String (Unique, Not Null)
- username: String (Unique, Not Null)
- password: String (암호화됨)
- profileImage: String (프로필 사진 URL)
- bio: String (자기소개)
- createdAt: LocalDateTime
- updatedAt: LocalDateTime
- isActive: Boolean (활성 여부)
- lastLoginAt: LocalDateTime
- loginAttempts: Integer (로그인 실패 횟수)
- lockedUntil: LocalDateTime (계정 잠금 시간)
```

## 6. API 엔드포인트 (초안)

### 인증 (공개)
- `POST /api/v1/auth/signup` - 회원가입
- `POST /api/v1/auth/login` - 로그인
- `POST /api/v1/auth/logout` - 로그아웃 (인증 필요)
- `POST /api/v1/auth/refresh` - 토큰 갱신

### 사용자 (공개)
- `GET /api/v1/users/{id}` - 사용자 프로필 조회
- `PUT /api/v1/users/{id}` - 프로필 수정 (인증 필요)
- `POST /api/v1/users/{id}/avatar` - 프로필 이미지 업로드 (인증 필요)
- `PUT /api/v1/users/{id}/password` - 비밀번호 변경 (인증 필요)

### 게시글 (공개 조회, 작성/수정/삭제는 인증 필요)
- `GET /api/v1/posts` - 게시글 리스트 조회 (공개)
- `GET /api/v1/posts/{id}` - 게시글 상세 조회 (공개)
- `POST /api/v1/posts` - 게시글 작성 (인증 필요)
- `PUT /api/v1/posts/{id}` - 게시글 수정 (인증 필요)
- `DELETE /api/v1/posts/{id}` - 게시글 삭제 (인증 필요)

## 7. 화면 설계 (초안)

### 메인 페이지 (공개)
- **헤더**: 로고, 검색창, 로그인/회원가입 버튼 (또는 사용자 메뉴)
- **게시글 리스트**:
  - 게시글 카드 (제목, 작성자, 작성일, 조회수, 댓글 수)
  - 정렬 버튼 (최신순, 인기순)
  - 페이지네이션
- **사이드바** (선택사항): 카테고리, 인기 게시글

### 회원가입 페이지
- 이메일 입력
- 사용자명 입력
- 비밀번호 입력 (보안 요구사항 표시)
- 비밀번호 확인 입력
- 약관 동의 체크박스
- 가입 버튼

### 로그인 페이지
- 이메일 입력
- 비밀번호 입력
- 자동로그인 체크박스
- 로그인 버튼
- 비밀번호 찾기 링크

### 프로필 페이지 (공개 조회)
- 프로필 사진
- 사용자명
- 자기소개
- 가입일
- 게시물 수 / 댓글 수
- 수정 버튼 (본인만)
- 사용자의 게시글 목록

## 8. 보안 고려사항
- [ ] 비밀번호는 절대 평문으로 저장하지 않음 (BCrypt)
- [ ] JWT 토큰에 만료 시간 설정 (Access Token: 1시간, Refresh Token: 7일)
- [ ] Refresh Token은 httpOnly 쿠키에 저장
- [ ] CORS 정책 설정
- [ ] SQL Injection 방지
- [ ] XSS 방지
- [ ] 비율 제한 (Rate Limiting) - 로그인: 5회/분, API: 100회/분
- [ ] 계정 잠금 메커니즘 (5회 실패 시 10분 잠금)
- [ ] 공개 게시글 API는 인증 불필요, 작성/수정/삭제 API는 인증 필수

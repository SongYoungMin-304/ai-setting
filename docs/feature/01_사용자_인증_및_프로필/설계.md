### 2.2 주요 클래스 설계

#### User Entity
```kotlin
@Entity
@Table(name = "users")
data class User(
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    val id: Long? = null,
    
    @Column(unique = true, nullable = false)
    val email: String,
    
    @Column(unique = true, nullable = false)
    val username: String,
    
    @Column(nullable = false)
    val password: String, // BCrypt 암호화
    
    val profileImage: String? = null,
    val bio: String? = null,
    
    @CreationTimestamp
    val createdAt: LocalDateTime = LocalDateTime.now(),
    
    @UpdateTimestamp
    val updatedAt: LocalDateTime = LocalDateTime.now(),
    
    val isActive: Boolean = true,
    val lastLoginAt: LocalDateTime? = null,
    val loginAttempts: Int = 0,
    val lockedUntil: LocalDateTime? = null,
    
    @OneToMany(mappedBy = "author", cascade = [CascadeType.REMOVE])
    val posts: List<Post> = emptyList(),
    
    @OneToMany(mappedBy = "author", cascade = [CascadeType.REMOVE])
    val comments: List<Comment> = emptyList()
)
```

#### Post Entity
```kotlin
@Entity
@Table(name = "posts")
data class Post(
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    val id: Long? = null,
    
    @Column(nullable = false)
    val title: String,
    
    @Column(columnDefinition = "TEXT", nullable = false)
    val content: String,
    
    @ManyToOne
    @JoinColumn(name = "author_id", nullable = false)
    val author: User,
    
    @CreationTimestamp
    val createdAt: LocalDateTime = LocalDateTime.now(),
    
    @UpdateTimestamp
    val updatedAt: LocalDateTime = LocalDateTime.now(),
    
    var viewCount: Int = 0,
    
    @OneToMany(mappedBy = "post", cascade = [CascadeType.REMOVE])
    val comments: List<Comment> = emptyList()
)
```

#### Comment Entity
```kotlin
@Entity
@Table(name = "comments")
data class Comment(
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    val id: Long? = null,
    
    @Column(columnDefinition = "TEXT", nullable = false)
    val content: String,
    
    @ManyToOne
    @JoinColumn(name = "post_id", nullable = false)
    val post: Post,
    
    @ManyToOne
    @JoinColumn(name = "author_id", nullable = false)
    val author: User,
    
    @CreationTimestamp
    val createdAt: LocalDateTime = LocalDateTime.now(),
    
    @UpdateTimestamp
    val updatedAt: LocalDateTime = LocalDateTime.now()
)
```

### 2.3 JWT 인증 플로우

```
1. 사용자 로그인 (POST /api/v1/auth/login)
   ├─ 이메일/비밀번호 검증
   ├─ BCrypt 암호 비교
   └─ JWT 토큰 생성
   
2. JWT 토큰 발급
   ├─ Access Token (만료: 1시간)
   ├─ Refresh Token (만료: 7일)
   └─ HttpOnly 쿠키에 저장
   
3. 인증된 요청
   ├─ Authorization: Bearer <AccessToken>
   ├─ JwtAuthenticationFilter에서 검증
   └─ SecurityContext에 사용자 정보 설정
   
4. 토큰 갱신 (POST /api/v1/auth/refresh)
   ├─ Refresh Token 검증
   └─ 새로운 Access Token 발급
```

```sql
CREATE TABLE users (

  password VARCHAR(255) NOT NULL,
  profile_image VARCHAR(500),
```

```sql
CREATE TABLE posts (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  title VARCHAR(255) NOT NULL,
  content LONGTEXT NOT NULL,
  author_id BIGINT NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  view_count INT DEFAULT 0,
  FOREIGN KEY (author_id) REFERENCES users(id) ON DELETE CASCADE,
  INDEX idx_author_id (author_id),
  INDEX idx_created_at (created_at)
);
```

#### comments 테이블
```sql
CREATE TABLE comments (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,

  author_id BIGINT NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (post_id) REFERENCES posts(id) ON DELETE CASCADE,
  FOREIGN KEY (author_id) REFERENCES users(id) ON DELETE CASCADE,
  INDEX idx_post_id (post_id),
  INDEX idx_author_id (author_id)
);
```

## 6. 보안 설계

### 6.1 JWT 토큰 구조
```
Header: {
  "alg": "HS256",
  "typ": "JWT"
}

Payload: {
  "sub": "user_id",
  "email": "user@example.com",
  "username": "username",
  "iat": 1629216000,
  "exp": 1629219600,  // 1시간
  "type": "ACCESS"
}

Signature: HMACSHA256(base64UrlEncode(header) + "." + base64UrlEncode(payload), secret)
```

### 6.2 토큰 저장 전략
```
- Access Token: localStorage (XSS 위험 있음, 하지만 편의성)
- Refresh Token: HttpOnly 쿠키 (CSRF 위험 있음, 하지만 보안성 높음)
```

### 6.3 CORS 정책
```kotlin
// SecurityConfig.kt
@Configuration
class CorsConfig {
  @Bean
  fun corsConfigurationSource(): CorsConfigurationSource {
    val configuration = CorsConfiguration()
    configuration.allowedOrigins = listOf("http://localhost:3000")
    configuration.allowedMethods = listOf("GET", "POST", "PUT", "DELETE", "OPTIONS")
    configuration.allowedHeaders = listOf("*")
    configuration.allowCredentials = true
    
    val source = UrlBasedCorsConfigurationSource()
    source.registerCorsConfiguration("/**", configuration)
    return source
  }
}
```

# 게시글 댓글 좋아요 기능 - 기획서

## 1. 기능 목적 및 배경

사용자가 다른 사용자의 댓글에 대해 공감을 표현할 수 있는 기능입니다. 댓글에 대한 긍정적 피드백을 제공하여 커뮤니티 활동을 활성화하고, 사용자 참여도를 높이는 것을 목표로 합니다.

**배경:**
- 게시글에 달린 댓글에 대한 평가 메커니즘 부재
- 사용자가 좋은 댓글에 대해 표현할 수 있는 방법 필요
- 커뮤니티 활성화 및 사용자 참여 증대

---

## 2. 요구사항 정의

### 2.1 기능적 요구사항

| 항목 | 요구사항 | 상세 내용 |
|------|---------|---------|
| 좋아요 추가 | 사용자가 댓글에 좋아요를 추가할 수 있어야 함 | 클릭으로 즉시 반영 |
| 좋아요 취소 | 이미 좋아요한 댓글의 좋아요를 취소할 수 있어야 함 | 토글 방식 |
| 좋아요 수 표시 | 댓글별 좋아요 수를 표시해야 함 | 실시간 업데이트 |
| 좋아요 여부 표시 | 사용자가 해당 댓글에 좋아요했는지 표시해야 함 | UI로 명확히 구분 |
| 중복 방지 | 한 사용자는 한 댓글에 한 번만 좋아요할 수 있어야 함 | 데이터 무결성 보장 |

### 2.2 비기능적 요구사항

- **성능**: 좋아요 요청은 1초 이내에 처리되어야 함
- **동시성**: 동시 요청에 대한 안정적인 처리 필요
- **확장성**: 향후 다른 엔티티(게시글, 사용자 등)에 확장 가능한 설계

## 3. 사용자 시나리오

### 시나리오 1: 좋은 댓글에 좋아요
```
1. 사용자 A가 게시글을 조회한다
2. 댓글 목록에서 좋은 댓글을 발견한다
3. 댓글의 좋아요 버튼을 클릭한다
4. 좋아요 수가 증가하고, 버튼 상태가 변경된다 (채움/비움)
5. 다른 사용자도 해당 댓글의 업데이트된 좋아요 수를 볼 수 있다
```

### 시나리오 2: 좋아요 취소
```
1. 사용자 A가 이전에 좋아요한 댓글을 다시 본다
2. 활성화된 좋아요 버튼을 클릭한다
3. 좋아요가 취소되고 수가 감소한다
4. 버튼 상태가 비활성화된다
```

### 시나리오 3: 댓글 조회 시 좋아요 정보 표시
```
1. 사용자가 게시글과 댓글을 조회한다
2. 각 댓글에 좋아요 수가 표시된다
3. 사용자가 이전에 좋아요한 댓글은 버튼 상태가 활성화된 상태로 표시된다
```

## 4. 범위 및 제약사항

### 4.1 포함 범위
- 댓글에 대한 좋아요 추가/취소 기능
- 좋아요 수 조회 및 표시
- 사용자별 좋아요 상태 추적
- API 엔드포인트 (POST, DELETE)
- 프론트엔드 UI 컴포넌트

### 4.2 제외 범위 (향후 고려)
- 싫어요 기능
- 좋아요 순으로 정렬
- 좋아요 알림 기능
- 좋아요 한 사용자 목록 조회

### 4.3 제약사항
- 로그인한 사용자만 좋아요 가능
- 댓글 작성자는 자신의 댓글에 좋아요할 수 없음 (검토 후 결정)
- 삭제된 댓글의 좋아요 정보는 함께 삭제됨

# 게시글 댓글 좋아요 기능 - 설계서

## 1. 기술 스택

- **Backend**: Spring Boot 3.x, JPA/Hibernate
- **Frontend**: React, TypeScript, Zustand (상태관리)
- **Database**: PostgreSQL
- **Cache**: Redis (좋아요 수 캐싱)
- **API**: RESTful API

---

## 2. 데이터베이스 설계

### 2.1 comment_likes 테이블

```sql
CREATE TABLE comment_likes (
    id BIGSERIAL PRIMARY KEY,
    comment_id BIGINT NOT NULL,
    user_id BIGINT NOT NULL,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    
    CONSTRAINT fk_comment_likes_comment 
        FOREIGN KEY (comment_id) 
        REFERENCES comments(id) ON DELETE CASCADE,
    
    CONSTRAINT fk_comment_likes_user 
        FOREIGN KEY (user_id) 
        REFERENCES users(id) ON DELETE CASCADE,
    
    CONSTRAINT uk_comment_user_unique 
        UNIQUE(comment_id, user_id)
);

CREATE INDEX idx_comment_likes_comment_id ON comment_likes(comment_id);
CREATE INDEX idx_comment_likes_user_id ON comment_likes(user_id);
```

### 2.2 comments 테이블 수정

```sql
ALTER TABLE comments 
ADD COLUMN IF NOT EXISTS like_count INT NOT NULL DEFAULT 0,
ADD COLUMN IF NOT EXISTS liked_at TIMESTAMP;
```

### 2.3 설계 고려사항
- `comment_id + user_id` 복합 유니크 키로 중복 방지
- CASCADE DELETE로 댓글/사용자 삭제 시 자동 정리
- like_count는 비동기 배치로 주기적 동기화
- 인덱스로 조회 성능 최적화

## 3. Backend API 설계

### 3.1 댓글에 좋아요 추가

**Endpoint**: `POST /api/v1/comments/{commentId}/likes`

**Request**:
```json
{
  // 요청 body 없음 (사용자는 JWT 토큰으로 식별)
}
```

**Response** (200 OK):
```json
{
  "success": true,
  "data": {
    "commentId": 123,
    "likeCount": 45,
    "isLiked": true,
    "likedAt": "2026-04-20T14:30:00Z"
  }
}
```

**Error Cases**:
- 401: 미인증 사용자
- 403: 자신의 댓글에 좋아요 (향후 결정)
- 404: 존재하지 않는 댓글
- 409: 이미 좋아요한 댓글

---

### 3.2 댓글 좋아요 취소

**Endpoint**: `DELETE /api/v1/comments/{commentId}/likes`

**Response** (200 OK):
```json
{
  "success": true,
  "data": {
    "commentId": 123,
    "likeCount": 44,
    "isLiked": false
  }
}
```

**Error Cases**:
- 401: 미인증 사용자
- 404: 존재하지 않는 댓글 또는 좋아요 기록

---

### 3.3 댓글 목록 조회 (좋아요 정보 포함)

**Endpoint**: `GET /api/v1/posts/{postId}/comments`

**Query Parameters**:
```
?page=0&size=20&sort=createdAt,desc
```

**Response** (200 OK):
```json
{
  "success": true,
  "data": {
    "content": [
      {
        "id": 123,
        "postId": 456,
        "userId": 789,
        "content": "좋은 댓글입니다",
        "likeCount": 45,
        "isLiked": true,
        "createdAt": "2026-04-20T10:00:00Z",
        "updatedAt": "2026-04-20T10:00:00Z"
      }
    ],
    "totalElements": 100,
    "totalPages": 5,
    "currentPage": 0
  }
}
```

## 4. Backend 구현 구조

### 4.1 계층 구조

```
Controller
  ↓
Service (트랜잭션, 비즈니스 로직)
  ↓
Repository (데이터 접근)
  ↓
Entity/Domain
```

### 4.2 주요 클래스

- `CommentLike`: JPA Entity
- `CommentLikeController`: REST 엔드포인트
- `CommentLikeService`: 좋아요 추가/취소 로직
- `CommentLikeRepository`: JPA Repository
- `CommentLikeDTO`: DTO (응답 데이터)

### 4.3 트랜잭션 관리
- 좋아요 추가/취소: `@Transactional`로 ACID 보장
- 중복 방지: DB 제약 조건 + 애플리케이션 로직 이중 검증

## 5. Frontend 컴포넌트 설계

### 5.1 CommentLikeButton 컴포넌트

```typescript
interface CommentLikeButtonProps {
  commentId: number;
  initialLikeCount: number;
  initialIsLiked: boolean;
}

interface LikeState {
  likeCount: number;
  isLiked: boolean;
  isLoading: boolean;
  error: string | null;
}
```

### 5.2 상태 관리 (Zustand)

```typescript
// useCommentLikeStore
- likeStates: Map<commentId, LikeState>
- toggleLike(commentId): Promise
- setLikeState(commentId, state)
```

### 5.3 UI 상태 흐름

```
idle 상태
  ↓ (사용자 클릭)
loading 상태 (요청 중)
  ↓
success 상태 (like_count 업데이트, 버튼 상태 토글)
  또는
error 상태 (토스트 메시지 표시)
  ↓
idle 상태 복귀
```

### 5.4 UI 상세

- **버튼 디자인**:
  - 활성 상태: 🧡 채워진 하트 + 숫자
  - 비활성 상태: 🤍 빈 하트 + 숫자
  
- **로딩 상태**: 버튼 disabled + 로딩 스피너
- **에러 처리**: 토스트 메시지 + 상태 롤백

## 6. 성능 및 보안

### 6.1 성능 최적화
- Redis 캐싱으로 like_count 빠른 조회
- DB 인덱스로 조회 성능 확보
- 배치 작업으로 비동기 동기화

- JWT 인증 필수 (로그인한 사용자만)

- Rate Limiting (좋아요 스팸 방지)
- SQL Injection 방지 (Prepared Statement)
